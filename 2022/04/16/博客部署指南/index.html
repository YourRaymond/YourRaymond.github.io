解决部署后图片不能正常显示的问题

  1. 打开修改Hexo配置，把hexo根目录文件夹里的全局配置文件_config.yml打开，修改post_asset_folder: false为 post_asset_folder: ture。这样在部署/新建一个引用了本地图片的md文件时，会在同级目录下创建一个与文件同名的文件夹。默认都是在hexo根目录\source\_posts下的。

  2. 如果之前Typora图片的存放路径选择的是复制图片到./${filename}.assets文件夹，并勾选了优先使用相对路径，并且已经基于此设置已经写好了md文件，那么可以在md文件中用Ctrl+H将./${filename}.assets替换为./${filename}，使得可以基于步骤3仍然正常显示图片。

  3. 修改Typora图片的存放路径为复制到指定目录./${filename}，防止每次部署到Hexo前都要做重复的更改，一劳永逸。

  4. 在hexo的根目录，右键Gir bash here，输入命令npm install hexo-asset-image --save，安装这个路径转换的插件。

  5. 但是安装完这个hexo-asset-image 插件仍不起作用，那么需要改写它。注意，这并不是说它不起作用就不安装它，因为接下来的该写需要在这个插件的配置中进行。
